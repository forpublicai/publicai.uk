# CEO Plan: publicai.uk — Live Tool-Call Architecture Demo

Generated by /plan-ceo-review on 2026-03-19
Branch: main | Mode: SCOPE EXPANSION
Repo: publicai.uk

## Vision

### 10x Check
The 10x version is a genuinely useful public service that's open today. Three working modes (BBC, justice, planning) powered by real public data feeds, a "founding institution" registration form capturing contacts, a live usage counter ("143 questions answered today") creating social proof and urgency, and shareable conversation permalinks — a DSIT official can forward a specific Q&A to their minister with one click.

### Platonic Ideal
A DSIT official opens /demo, picks "BBC Companion," types "What's happening with NHS reform?" — the right panel lights up: `→ bbc_news_search("NHS reform")... ✓ 3 articles found`. The answer arrives in ~3 seconds, clean and sourced. Below it: "This answer was assembled via structured tool calls to BBC News. No content was stored. The BBC controls every piece of content served." They feel: this is real, this is safe for the BBC, I should write that letter. That emotional arc — curiosity → comprehension → urgency — is the product.

## Accepted Scope (added to this plan)

- `/demo` page with full two-panel layout (chat left, tool trace right)
- Three mode switcher: BBC Companion / Justice Guide / Planning Assistant
- Per-mode suggested prompt chips (3-4 per mode, pre-fill and submit on click)
- `/api/chat` route with Claude API via **Vercel AI SDK** + tool-use:
  - `bbc_news_search` → BBC RSS feeds (feeds.bbci.co.uk)
  - `gov_guidance_search` → GOV.UK search API (gov.uk/api/search.json)
  - `legislation_search` → legislation.gov.uk API (justice mode)
- Streaming tool call trace panel with real-time tool invocation display
- Every answer shows citations with source name + URL
- Vercel AI SDK (`ai` + `@ai-sdk/anthropic`) installed as dependency
- Client-side 10-question session limit (no Upstash needed)
- 'Request a briefing' CTA at bottom of /demo (Formspree or mailto)
- Hero copy rewrite: replace 'fake-but-concrete' with 'working prototype' framing
- Main page prominent link/button to /demo
- Shareable conversation permalinks via base64 query-param
- Error handling: all tool execute() wrapped in try/catch; ANTHROPIC_API_KEY validated at startup

### Mode Switcher
- Style: underline tabs — `text-xs font-semibold uppercase tracking-[0.2em]`
- Active: 2px `#f0442c` underline, `text-stone-900`
- Inactive: `text-stone-500`, no underline, hover `text-stone-700`
- No background fill — matches editorial vocabulary of existing site

### Interaction States

| Feature | Loading | Empty | Error | Success | Session limit |
|---------|---------|-------|-------|---------|--------------|
| Chat | Animated ●●● dots in stone-900 bubble | Mode-specific placeholder + 4 prompt chips | Red-tinted bubble, "Something went wrong — try again", retry on tap | Message + footnote citations | Disabled input, stone overlay, "10-question limit reached" message |
| Tool trace | `→ tool_name("args")…` with pulse animation | "Ask a question to see how Public AI works…" | Tool error shown as `✗ tool_name · error`, stream continues | `✓ tool_name · N results · Source` | N/A |
| Mode switcher | — | — | — | Active underline | — |
| Prompt chips | Disabled (opacity-50) while loading | — | — | Click pre-fills + submits | Hidden after session limit |

### Colour System for New Components

- Tool call `→` (in-flight): `text-stone-600`, pulse animation
- Tool call `✓` (result): `text-stone-700` — no green, keeps palette clean
- Tool call `✗` (error): `text-red-500`
- Active mode underline: `#f0442c` (2px border-bottom)
- Monospace font: `font-mono` (system mono — no new font dep)

### Responsive / Mobile (< 768px)

- Two-panel collapses to single column
- Chat panel: full width, on top
- Tool trace: collapses to `"How it works ▾"` toggle, expands on tap
- Mode switcher: stays as inline text tabs (wraps if needed)
- Suggested prompts: horizontal scroll or 2×2 grid
- Briefing CTA: stacks vertically

### Accessibility

- Mode switcher: `role="tablist"`, each tab `role="tab"` + `aria-selected`
- Chat input: `aria-label="Ask a question"`, `aria-describedby` → mode label
- Send button: `aria-label="Send question"`
- Message list: `aria-live="polite"` (announces new messages to screen readers)
- Tool trace: `aria-live="off"` (supplementary — doesn't interrupt screen reader)
- Session limit: `aria-label="10 question limit reached"` on disabled input
- Share button: `aria-label="Copy link to this conversation"`
- Touch targets: min 44×44px for all interactive elements
- Keyboard: arrow keys between mode tabs, Tab to input, Enter to send
